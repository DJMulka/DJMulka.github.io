Package Management Automation

Duration: 1 year
Role: Lead Developer
Categoty: Internal Tool
Status: Patent-Pending

Project Overview

The Package Manager Tool is a critical software utility I developed to solve a major bottleneck in scaling augmented reality work instruction deployments across multiple systems. LightGuide's core platform enables users to create detailed, interactive work instructions using XML-based workflows with embedded assets like graphics, text, vision tools, and hardware commands. However, moving these workflows between stations with differing configurations led to asset mismatches and calibration errors.

To address this, I built a robust packaging system that automates the collection, export, and import of workflows and their associated assets. This tool ensures consistent behavior across varying hardware setups and has become essential for customers scaling AR solutions across production lines with non-identical systems.

Technical Implementation

VB.NET UI/UX Development File Packaging & Compression Asset Dependency Resolution System Calibration Scalable Workflow Distribution

Key Features

Automated Asset Collection: Implements a custom algorithm to parse a work instruction XML file, identify all referenced assets, and collect them into a single exportable package.

Custom Package Format with Metadata: Packages all assets into a ZIP archive with a custom file extension and icon, making it easy to recognize, share, and import across systems.

Intelligent Recalibration Utility: Upon import, users are guided through a recalibration step that aligns graphics to the new station's hardware setup,ensuring accuracy even across non-identical systems.

Scalability for Multi-Station Deployments: Supports mass deployment of shared work instructions, reducing authoring overhead for customers rolling out AR systems across full assembly lines.

Results & Impact

The implementation delivered measurable improvements across multiple key performance indicators:

  • Simplified Workflow Portability
  • Improved Deployment Scalability
  • Reduced Configuration Errors

Challenges & Solutions

Challenge: Workflows referenced numerous distributed assets that weren't bundled together, causing transfer issues.

Solution: Designed and implemented a dependency-wrangling algorithm that recursively identifies and collects all required assets starting from a root instruction file.

Challenge: Hardware variations between stations made it difficult to maintain consistent visual alignment.

Solution: Introduced a recalibration process on import, aligning graphics based on the new station's calibration to preserve visual fidelity and spatial accuracy.

Challenge: Customers needed a way to deploy instructions at scale without manual reconfiguration.

Solution: Created a standardized, portable package format that could be dropped into any station and executed without additional setup or asset alignment.
